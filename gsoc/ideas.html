IDEA 7: Cytoscape plugin for importing and visualizing variant data

Background: Variant data identifies genomic regions where individual base pairs are different from a reference genome. These variations (e.g., SNPs) can influence gene regulation, structure and function, and are thus associated many genetic diseases. These data sets are quick large and difficult to navigate.

Goal: To make a Cytoscape plugin that reads in variant data files and dynamically maps the data to biological networks and pathways independently loaded into Cytoscape. The plugin should provide the databases needed to perform the mapping as well as default visualization settings, such as coloring all genes with variants in coding regions red.

The plugin should:

  • Be able to read .vcf files (http://www.1000genomes.org/node/101). In fact, there is an indexed and compressed form of .vcf files that we would ideally work with.

  • Recognize the mapping of Ensembl IDs to genomic coordinates. Pathway and networks will already be mapped to Ensembl via GenMAPP-CS plugins.

  • Identify the subset of .vcf data rows to import based on the genomic coordinates of the pathway or network nodes.

  • Import these rows. When more than one row maps to a single gene, a metanode should be created.

  • Parse important values from the INFO column of .vcf files and create separate CyAttributes.

  • Define various default visual color or size mappings based on the values of the imported .vcf attributes.


Language and Skills: Java/Swing

Potential Mentors: Alex Pico, Paul Tang, Nathan Salomonis, Michael Heuer

IDEA 28: User Interaction on Biomedical Linked Data

The idea for this year’s project is to go one step further and to allow the use of Cytoscape to visually define queries in SPARQL. The basic idea is that from a graph, a sub-graph can be defined where a few nodes or edges can be marked as variables (and eventually filters can be associated to them): this is equivalent to a (simple) SPARQL query.   • Implementing heuristics for query-selection: in the current plugin, queries can be performed on demand, but they are also associated to specific actions. For instance, right-selecting a node of type protein can present as an option to expend the graph with the results of a query which is can include the specific protein as a value. This is possible through a simple markup through which queries are annotated by the user. Such markup could be included in the visual query/construction process and/or extracted from simple query patterns.

Some of the ideas in this project have been already prototyped, so we have a good understanding of the difficulty and timing required.

This project has the potential to make Cytoscape an ideal tool to navigate and query Linked Data resources (both in biomedical and, more generally, in Open Data domains).

IDEA 29: CentiScaPe for multiple networks analysis and graphical output enhancement

CentiScaPe allows identifying network nodes that are relevant from both experimental and topological viewpoints. CentiScaPe also provides a Boolean logic- based tool that allows easy characterization of nodes whose topological relevance depends on more than one centrality.   • The main framework for the results analysis is the Abstract Interpretation theory, particularly the interval analysis (and eventually others) [1].

Idea 30: Porting Interference App to Cytoscape 3 with new features for undirected and directed networks

Background: Interference is a Cytoscape plug-in for virtual knock-out experiments on complex networks. It allows to evaluate the topological effects of single or multiple nodes removal from a network. In this perspective, interference allows virtual node knock-out experiments: it is possible to remove one or more nodes from a network and analyze the consequences on network structure, by looking to the variations of the node centralities values. As the centrality value of a node is strictly dependent on the network structure and on the properties of other nodes in the network, the consequences of a node deletion are well captured by the variation on the centrality values of all the other nodes. With the interference app the user can:

  • Simulate pharmacological treatment: one can potentially predict side effects of the drug by looking at the topological properties of nodes in a drug-treated network, meaning with that a network in which a drug-targeted node (protein) was removed. To inhibit a protein (for instance a kinases) corresponds to removing the node from the network

  • Simulate gene deletion: gene deletion implies losing encoded proteins, thus resulting in the corresponding removal of one or more nodes from a protein network

Goal: The Interference plugin is based on some features of the CentiScaPe plugin, and is available for Cytoscape 2.8.x but not yet for Cytoscape 3. The IDEA is to port the plugin to Cytoscape 3 and to add some features. The plugin works for undirected networks, but now a new version of the CentiScaPe plugin allows to compute several centralities also for directed networks. The new tool should:

  • Work with Cytoscape 3.

  • Provide the Eigenvector, Bridging Centrality and Edge Betweenness

  • centrality Interference (now missing).

  • Provide the directed Interference for all the centralities implemented

  • More info at: http://www.cbmc.it/%7Escardonig/interference/Interference.php


Language and Skills: Java

Potential Mentors: Giovanni Scardoni (CBMC, University of Verona),Fausto Spoto (University of Verona)

Idea 32: Develop Heat-map Based Data Views for Cytoscape 3 using CoolMap

Background: CoolMap is a novel omics data exploration application/library that offers new ways to explore complex molecular data. It is based on the classic heatmap paradigm, but allows users to use generated (such as clustering) or pre-defined (such as gene ontology or sample groups) ontologies to aggregate big datasets into concept-level, human manageable views. The rows and columns of CoolMap can be expanded/collapsed interactively and each cell can be plotted in many ways (color gradient, statistical plots, etc.). It efficiently facilitates the exploratory data analysis, and help researchers generate hypotheses from data.

Paired with other software like Cytoscape, this visualization duo has a lot of potential to transform the knowledge discovery processes from biomedical networks. CoolMap can be a good upgrade candidate for the currently used Java TreeView for Cytoscape.


A basic Cytoscape to CoolMap plugin framework will be provided to the student. The student then can harness the power of Cytoscape and CoolMap to explore new visualization possibilities.


CoolMap will public in several weeks and news and updates can be found at http://www.coolmap.org


Goal: Develop either a Cytoscape plugin using CoolMap’s core visualization library, or a CoolMap plugin that utilizes the full functionality of the CoolMap application to enhance Cytoscape’s heatmap-based visualization capabilities.
